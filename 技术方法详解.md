# 输电杆塔仿真计算模型自动生成技术研究方案

## 1 技术研究总体方案

### 1.1 项目背景与意义

输电杆塔是电力传输的基础设施，其结构设计复杂、尺寸关系多样。当前，杆塔仿真计算模型多数由人工基于二维工程图纸手工建立，该过程存在以下主要问题：

1. **效率低下**：手工建模耗时长，需要多人次协作验证
2. **易出错**：坐标编输错误、杆件关联遗漏等人为错误难以避免
3. **无法溯源**：模型与原始图纸的对应关系不清晰，难以追踪修改
4. **可复用性差**：不同规格杆塔需重复建模工作

本研究基于杆塔二维工程图纸（TXT格式坐标数据），研发自动转换为三维仿真计算模型（Excel/JSON格式）的系统，实现：

- **全自动坐标转换**：从2D平面图纸自动生成3D空间坐标
- **多视图融合**：支持双视图（正面+侧面）、单视图（仅正面）等多种输入格式
- **智能拼接**：自动识别杆件拓扑关系，准确拼接塔体与塔头
- **模型验证**：输出规范化节点编号、杆件截面、材质属性等完整仿真参数

## 2 关键技术与核心算法

### 2.1 二维坐标向三维坐标转换方法

#### 2.1.1 坐标系统定义

在二维图纸中，使用两视图投影表示：
- **正面图（Front View）**：采用 X-Z 坐标系（X为水平，Z为竖直）
- **侧面图（Side/Overhead View）**：采用 Y-Z 坐标系（Y为水平，Z为竖直）

三维世界坐标系定义为：
$$\mathbf{p}_{3D} = (X, Y, Z) \in \mathbb{R}^3$$

其中：
- $X$：水平左右方向（从正面看）
- $Y$：水平前后方向（从侧面看）
- $Z$：竖直向上方向（公共轴）

#### 2.1.2 基本投影关系

对于杆件两个端点在双视图中的坐标：
- 正面图：$P_{front} = (X, Z)$
- 侧面图：$P_{side} = (Y, Z)$

则三维坐标通过组合投影直接获得：
$$X_{3D} = X_{front}, \quad Y_{3D} = Y_{side}, \quad Z_{3D} = Z_{front} = Z_{side}$$

**核心约束**：Z坐标在两视图中必须一致，否则表示数据不符合投影关系，需要进行异常检测与修正。

#### 2.1.3 算法实现步骤

**核心算法：`reconstruct3d_front()` 和 `reconstruct3d_right()`**

```python
# 伪代码表示转换逻辑
def reconstruct3d_dual_view(front_coords, right_coords):
    """
    将双视图坐标转换为3D坐标
    
    输入：
        front_coords: Dict[str, [(x1,z1), (x2,z2)]] - 正面图坐标对
        right_coords: Dict[str, [(y1,z1), (y2,z2)]] - 侧面图坐标对
    
    输出：
        coords_3d: Dict[str, [(x,y,z), (x,y,z)]] - 三维坐标对
    """
    coords_3d = {}
    
    for gid in front_coords:
        if gid not in right_coords:
            continue  # 杆件必须同时出现在两视图
        
        front_seg = front_coords[gid]  # [(x1,z1), (x2,z2)]
        right_seg = right_coords[gid]   # [(y1,z1), (y2,z2)]
        
        # 提取各坐标分量
        x1, z1_f = front_seg[0]
        x2, z2_f = front_seg[1]
        y1, z1_r = right_seg[0]
        y2, z2_r = right_seg[1]
        
        # Z值验证：两视图的Z值必须对应相等
        # （允许小的容差用于浮点数比较）
        if not isclose(z1_f, z1_r, tol=1e-6) or not isclose(z2_f, z2_r, tol=1e-6):
            # 记录错误或采用策略修正
            z1 = (z1_f + z1_r) / 2.0
            z2 = (z2_f + z2_r) / 2.0
        else:
            z1 = z1_f
            z2 = z2_f
        
        # 组合三维坐标
        p1_3d = (x1, y1, z1)
        p2_3d = (x2, y2, z2)
        coords_3d[gid] = [p1_3d, p2_3d]
    
    return coords_3d
```

**关键步骤**：
1. **坐标读取**：从2D图纸数据中提取X、Y、Z分量
2. **Z值对齐**：验证两视图的Z坐标是否一致，不一致时采用平均或优先级策略
3. **组合生成**：将X、Y、Z分量组合成三维坐标向量
4. **有效性检验**：验证生成的3D坐标的物理合理性

#### 2.1.4 坐标中心化与平移修正

**问题**：原始图纸坐标系的原点通常不在杆塔的物理中心，需要进行坐标变换以便于计算和可视化。

**算法**：

```python
def center_and_translate_model(coords_3d_front, coords_3d_right):
    """
    1. 计算模型几何中心
    2. 平移模型使其中心化
    """
    # 步骤1：收集所有端点
    all_points = []
    for seg in coords_3d_front.values():
        all_points.extend(seg)
    for seg in coords_3d_right.values():
        all_points.extend(seg)
    
    # 步骤2：计算中心与最低点
    xs = [p[0] for p in all_points]
    ys = [p[1] for p in all_points]
    zs = [p[2] for p in all_points]
    
    x_center = (min(xs) + max(xs)) / 2.0
    y_center = (min(ys) + max(ys)) / 2.0
    z_min = min(zs)
    
    center = (x_center, y_center, 0.0)
    
    # 步骤3：平移所有坐标
    dx, dy, dz = -x_center, -y_center, -z_min
    
    def translate_seg(seg):
        return [(p[0]+dx, p[1]+dy, p[2]+dz) for p in seg]
    
    translated_front = {k: translate_seg(v) for k, v in coords_3d_front.items()}
    translated_right = {k: translate_seg(v) for k, v in coords_3d_right.items()}
    
    return translated_front, translated_right
```

### 2.2 面向不同视图组合的三维坐标解算方法

#### 2.2.1 双视图模式（A线：正侧面）

**适用场景**：标准塔体结构，拥有完整的正面图与侧面图两份视图。

**特点**：
- 信息最完整，三维坐标直接通过投影组合获得
- Z坐标冗余，用于数据一致性校验
- 支撑杆、横向杆、X型杆等各类构件均有明确投影

**解算流程**：

1. **视图识别与分类**

   ```python
   def classify_members_dual_view(view_coords):
       """
       将图纸中的杆件分为三类：
       - 支撑杆：ID以01/02/03/04结尾，贯穿塔身高度
       - 横向杆：近似水平（|Δy| ≤ 容差）
       - X型杆：斜交，用于增加刚度
       """
       supports = find_supports(view_coords)      # 长杆件
       horizontals = find_horizontals(view_coords, tol=容差)  # 水平杆件
       x_members = view_coords - supports - horizontals  # X型杆
       
       return supports, horizontals, x_members
   ```

2. **支撑杆斜率匹配与统一**

   由于扫描或输入误差，两视图中的支撑杆斜率可能略有偏差。算法采用**跨视图斜率约束**：

   ```python
   def match_support_slopes(front_supports, right_supports):
       """
       将两视图的支撑杆斜率统一到一致值
       
       策略：
       1. 分别计算两视图各支撑杆的斜率 k = Δy/Δx
       2. 对所有斜率求平均值或加权平均
       3. 重新调整端点坐标，保持斜率一致
       """
       front_slopes = [slope(seg) for seg in front_supports.values()]
       right_slopes = [slope(seg) for seg in right_supports.values()]
       
       # 采用策略：mean / weighted_mean / median
       unified_slope = weighted_mean(front_slopes + right_slopes)
       
       # 重新调整端点，保持斜率不变但位置更新
       for gid in front_supports:
           p1, p2 = front_supports[gid]
           # 保持中点，调整端点使斜率为 unified_slope
           mid_x = (p1[0] + p2[0]) / 2
           mid_y = (p1[1] + p2[1]) / 2
           # ... 重新计算端点
   ```

3. **顶横杆定位与等长约束**

   **原理**：塔顶处的横向杆件应该：
   - 位于两视图支撑杆的顶部交点处
   - 在两视图中长度相等

   ```python
   def align_top_span(front_support_models, right_support_models, 
                      front_horizontals, right_horizontals):
       """
       将两视图的顶横杆对齐到支撑杆顶部，并强制等长
       """
       # 1. 计算正面图支撑杆在顶部(y=y_max)的左右极值
       y_top = max_y_of_supports(front_support_models)
       left_f, right_f = extreme_x_at_y(front_support_models, y_top)
       
       # 2. 计算侧面图支撑杆在顶部的左右极值
       left_r, right_r = extreme_x_at_y(right_support_models, y_top)
       
       # 3. 两视图的顶横杆都固定到这四个端点
       front_horizontals[top_key] = [(left_f, y_top), (right_f, y_top)]
       right_horizontals[top_key] = [(left_r, y_top), (right_r, y_top)]
   ```

4. **其余横杆成对统一与投影**

   ```python
   def correct_paired_horizontals(front_h, right_h, 
                                  front_supports, right_supports):
       """
       除顶横杆外的其他横向杆件：
       1. 在高度上成对统一（两视图同高）
       2. 端点投影到各视图的支撑杆上
       """
       # 配对高度
       front_heights = sorted([avg_y(seg) for seg in front_h.values()])
       right_heights = sorted([avg_y(seg) for seg in right_h.values()])
       
       # 成对同高
       for i, (h_f, h_r) in enumerate(zip(front_heights, right_heights)):
           unified_h = (h_f + h_r) / 2.0
           
           # 投影到支撑杆
           left_f, right_f = extreme_x_at_y(front_supports, unified_h)
           left_r, right_r = extreme_x_at_y(right_supports, unified_h)
           
           # 更新坐标
           front_h[front_keys[i]] = [(left_f, unified_h), (right_f, unified_h)]
           right_h[right_keys[i]] = [(left_r, unified_h), (right_r, unified_h)]
   ```

5. **X型杆件匹配与坐标修正**

   **问题**：X型杆件在两视图中可能存在端点不对齐、长度不一致等问题。

   **解决方案**：建立X型对配对关系，并通过几何约束强制对齐。

   ```python
   def match_and_snap_x_pairs(front_x_dict, right_x_dict, 
                              front_supports, right_supports):
       """
       1. 将正面的斜杆与侧面的斜杆进行配对
       2. 通过几何约束（交点、平行度等）修正端点
       """
       # 配对：根据Y坐标重叠度
       front_pairs = extract_x_pairs(front_x_dict)
       right_pairs = extract_x_pairs(right_x_dict)
       matched = match_x_pairs(front_pairs, right_pairs)
       
       for (i_f, i_r) in matched:
           if i_f is None or i_r is None:
               continue
           
           # 强制对齐到支撑杆交点
           # 使用透视投影或等腰梯形约束
           snap_x_pair_to_supports(front_pairs[i_f], right_pairs[i_r],
                                  front_supports, right_supports)
   ```

**输出**：规范化的双视图坐标，满足投影一致性与几何约束。

#### 2.2.2 单视图模式（B线：仅正面图）

**适用场景**：塔头、地线支架等非塔体结构，通常仅有一份正面图。

**挑战**：
- 信息不完整，无法直接获得Y坐标
- 需要根据图纸的拓扑结构与设计规范进行推断
- 可能包含编码型坐标（如 ID编码法）

**转换流程**：

1. **一类节点处理（真实3D坐标）**

   定义：所有三个坐标 X、Y、Z 均为真实数值的节点。

   ```python
   def process_class1_nodes(nodes_raw):
       """
       一类节点直接使用，无需推算
       """
       class1_nodes = []
       for node in nodes_raw:
           x, y, z = node['X'], node['Y'], node['Z']
           if all(isinstance(c, float) for c in [x, y, z]):
               class1_nodes.append(node)
       return class1_nodes
   ```

2. **二类节点处理（参考型坐标）**

   定义：两个坐标是节点ID编码，一个坐标是真实值的节点。

   编码规则：节点 ID 在某坐标维度的值表示为 `10000 + base_node_id`，表示该坐标是参考的。

   ```python
   def process_class2_nodes(nodes_raw, class1_map):
       """
       二类节点需要基于参考杆件进行插值解算
       
       算法：
       1. 解析二类节点的引用关系（哪两个坐标是编码，哪个是真实）
       2. 从编码中提取参考杆件ID
       3. 在参考杆件上进行线性插值
       """
       class2_nodes = []
       
       for node in nodes_raw:
           node_id = node['node_id']
           x, y, z = node['X'], node['Y'], node['Z']
           
           # 识别编码坐标
           ref_coords = []  # 记录编码坐标的维度
           real_coord = None
           real_dim = None
           
           if isinstance(x, int) and x >= 10000:
               ref_coords.append(('X', x))
           else:
               real_coord = x
               real_dim = 'X'
           
           if isinstance(y, int) and y >= 10000:
               ref_coords.append(('Y', y))
           else:
               real_coord = y
               real_dim = 'Y'
           
           if isinstance(z, int) and z >= 10000:
               ref_coords.append(('Z', z))
           else:
               real_coord = z
               real_dim = 'Z'
           
           # 必须恰好有一个真实坐标
           if len(ref_coords) != 2:
               continue
           
           # 提取参考杆件ID
           ref_id_1 = (ref_coords[0][1] - 10000)
           ref_id_2 = (ref_coords[1][1] - 10000)
           
           # 从参考杆件中获取端点
           if ref_id_1 not in class1_map or ref_id_2 not in class1_map:
               continue  # 跳过无法解析的节点
           
           p1 = class1_map[ref_id_1]  # (x1, y1, z1)
           p2 = class1_map[ref_id_2]  # (x2, y2, z2)
           
           # 线性插值：求参数 t 使得约束坐标满足
           if real_dim == 'X':
               # 参考杆件在X方向变化，求使 X == real_coord 的 t
               if abs(p2[0] - p1[0]) < 1e-9:
                   continue  # 杆件在该方向无变化，无法插值
               t = (real_coord - p1[0]) / (p2[0] - p1[0])
           elif real_dim == 'Y':
               if abs(p2[1] - p1[1]) < 1e-9:
                   continue
               t = (real_coord - p1[1]) / (p2[1] - p1[1])
           else:  # real_dim == 'Z'
               if abs(p2[2] - p1[2]) < 1e-9:
                   continue
               t = (real_coord - p1[2]) / (p2[2] - p1[2])
           
           # 检查 t 范围，避免过度推算
           if t < -0.2 or t > 1.2:
               continue
           
           # 计算三维坐标
           new_x = p1[0] + t * (p2[0] - p1[0])
           new_y = p1[1] + t * (p2[1] - p1[1])
           new_z = p1[2] + t * (p2[2] - p1[2])
           
           class2_nodes.append({
               'node_id': node_id,
               'X': new_x,
               'Y': new_y,
               'Z': new_z
           })
       
       return class2_nodes
   ```

3. **迭代求解策略**

   由于二类节点可能相互引用，需要采用迭代策略：

   ```python
   def iterative_resolve_class2_nodes(nodes_raw):
       """
       多轮迭代求解所有二类节点
       """
       class1_nodes = extract_class1(nodes_raw)
       node_map = {n['node_id']: n for n in class1_nodes}
       
       unresolved = [n for n in nodes_raw if is_class2(n)]
       
       iteration = 0
       while unresolved and iteration < 10:
           iteration += 1
           newly_resolved = []
           
           for node in unresolved:
               result = try_resolve_class2(node, node_map)
               if result:
                   newly_resolved.append(result)
                   node_map[result['node_id']] = result
           
           if not newly_resolved:
               break  # 无进展，停止迭代
           
           unresolved = [n for n in unresolved if n not in newly_resolved]
       
       return list(node_map.values())
   ```

4. **对称性展开**

   若原始图纸中仅给出一个象限或半部的坐标，需根据对称性标记展开：

   ```python
   def expand_nodes_by_symmetry(base_nodes):
       """
       根据对称性标记生成对称节点
       
       对称类型：
       - 1: 关于Y轴对称（左右对称）
       - 2: 关于X轴对称（前后对称）
       - 3: 关于Z轴对称（180°旋转）
       - 4: 关于中心点对称
       """
       expanded = {}
       
       for node in base_nodes:
           nid = node['node_id']
           sym = node.get('symmetry_type', 0)
           x, y, z = node['X'], node['Y'], node['Z']
           
           # 原节点
           expanded[nid] = node
           
           # 对称节点
           if sym in (1, 4):
               # Y轴对称：X反号
               sym_node = node.copy()
               sym_node['node_id'] = nid + 1
               sym_node['X'] = -x
               expanded[nid + 1] = sym_node
           
           if sym in (2, 4):
               # X轴对称：Y反号
               sym_node = node.copy()
               sym_node['node_id'] = nid + 2
               sym_node['Y'] = -y
               expanded[nid + 2] = sym_node
           
           if sym in (3, 4):
               # 中心对称：X、Y反号
               sym_node = node.copy()
               sym_node['node_id'] = nid + 3
               sym_node['X'] = -x
               sym_node['Y'] = -y
               expanded[nid + 3] = sym_node
       
       return expanded
   ```

#### 2.2.3 多模型拼接与融合

**场景**：当处理塔体（A线）与塔头（B线）时，需要在空间中准确拼接两个独立的3D模型。

**算法**：

```python
def splice_dual_and_single_models(model_a_nodes, model_a_members,
                                   model_b_nodes, model_b_members):
    """
    塔体（A）与塔头（B）的拼接：
    1. 识别拼接特征点（通常是B线底部与A线顶部的对接面）
    2. 通过特征点匹配确定相对位置关系
    3. 变换B坐标到全局坐标系
    4. 合并节点与杆件
    """
    
    # 步骤1：在A线模型中寻找拼接候选点
    # 通常是Z值最大的横向杆（顶部）
    a_top_horizontal = find_top_horizontal(model_a_nodes)
    a_bridge_points = [
        a_top_horizontal.node1_id,
        a_top_horizontal.node2_id
    ]
    a_bridge_coords = [
        model_a_nodes[pid]['coordinate'] 
        for pid in a_bridge_points
    ]
    
    # 步骤2：在B线模型中寻找拼接候选点
    # 通常是Z值最小的横向杆（底部）
    b_bottom_horizontal = find_bottom_horizontal(model_b_nodes)
    b_bridge_points = [
        b_bottom_horizontal.node1_id,
        b_bottom_horizontal.node2_id
    ]
    b_bridge_coords = [
        model_b_nodes[pid]['coordinate'] 
        for pid in b_bridge_points
    ]
    
    # 步骤3：特征点匹配与变换估计
    # 使用最小二乘法或ICP算法
    transform = estimate_transformation(a_bridge_coords, b_bridge_coords)
    
    # 步骤4：应用变换到B模型的所有节点
    model_b_nodes_transformed = apply_transform(model_b_nodes, transform)
    
    # 步骤5：融合节点与杆件
    # 处理重复节点（在拼接界面处）
    merged_nodes = merge_nodes(model_a_nodes, model_b_nodes_transformed)
    merged_members = merge_members(model_a_members, model_b_members)
    
    return merged_nodes, merged_members
```

```python
    - 边长：100mm
```

#### 2.3.4 节点编号标准化

**问题**：自动生成的节点编号可能不在1-500的标准范围内，需要进行重新编号。

```python
def normalize_node_ids(nodes, members):
    """
    将节点编号标准化到1-500范围
    
    步骤：
    1. 收集现有有效编号
    2. 识别超出范围的编号
    3. 分配新编号（优先填补空缺）
    4. 更新所有引用
    """
    
    # 步骤1：收集有效编号的前缀
    valid_prefixes = set()
    for node in nodes:
        node_id = str(node['node_id'])
        prefix = node_id[:-1]  # 去掉最后一位
        try:
            num = int(prefix)
            if 1 <= num <= 500:
                valid_prefixes.add(prefix)
        except:
            pass
    
    # 步骤2：识别需要修改的节点
    mapping = {}  # 旧前缀 -> 新前缀
    for node in nodes:
        node_id = str(node['node_id'])
        prefix = node_id[:-1]
        
        try:
            num = int(prefix)
            if not (1 <= num <= 500):
                # 需要重编号
                mapping[prefix] = find_next_available_id(valid_prefixes)
                valid_prefixes.add(mapping[prefix])
        except:
            mapping[prefix] = find_next_available_id(valid_prefixes)
            valid_prefixes.add(mapping[prefix])
    
    # 步骤3：应用映射到节点
    for node in nodes:
        node_id = str(node['node_id'])
        prefix = node_id[:-1]
        suffix = node_id[-1]
        if prefix in mapping:
            node['node_id'] = mapping[prefix] + suffix
    
    # 步骤4：应用映射到杆件
    for member in members:
        for attr in ['node1_id', 'node2_id']:
            node_id = str(member[attr])
            prefix = node_id[:-1]
            suffix = node_id[-1]
            if prefix in mapping:
                member[attr] = mapping[prefix] + suffix
    
    # 步骤5：处理type=12的参考型节点
    for node in nodes:
        if node.get('node_type') == 12:
            for coord in ['X', 'Y', 'Z']:
                ref_id = node.get(coord)
                if isinstance(ref_id, str) and ref_id.startswith('1'):
                    # 解码：去掉开头的1
                    original_id = ref_id[1:]
                    prefix = original_id[:-1]
                    suffix = original_id[-1]
                    if prefix in mapping:
                        # 重新编码
                        new_id = '1' + ('0' * (4 - len(mapping[prefix]) - 1)) + mapping[prefix] + suffix
                        node[coord] = new_id
    
    return nodes, members, mapping
```

```python
def check_geometric_validity(nodes, members):
    """

    checks = {
        "all_members_have_endpoints": True,
        "no_zero_length_members": True,
        "no_nans_or_infs": True,
        "reasonable_coordinate_range": True
    }
    
    for member in members:
        n1_id = member['node1_id']
        n2_id = member['node2_id']
        
        if n1_id not in nodes or n2_id not in nodes:
            checks["all_members_have_endpoints"] = False
            continue
        
        p1 = nodes[n1_id]
        p2 = nodes[n2_id]
        
        # 检查零长杆件
        dist = euclidean_distance(p1, p2)
        if dist < 1e-6:
            checks["no_zero_length_members"] = False
        
        # 检查NaN/Inf
        for p in [p1, p2]:
            for coord in p:
                if not (-1e6 < coord < 1e6):
                    checks["no_nans_or_infs"] = False
                    checks["reasonable_coordinate_range"] = False
    
    return all(checks.values())
```

#### 2.5.2 关键点坐标精确提取

**问题**：扫描图纸或手工输入坐标时，可能存在误差（1-2像素或更大）。

**解决方案**：通过关键点检测与精细化：

```python
def refine_endpoint_coordinates(raw_endpoints, neighboring_elements, 
                                tolerance=0.5):
    """
    基于邻近图元的拓扑关系精细化端点坐标
    
    原理：
    1. 检测多条杆件的公共端点（原始坐标接近）
    2. 计算这些杆件的精确交点
    3. 更新所有相关端点到交点位置
    """
    
    # 步骤1：聚类：找出接近的端点
    clusters = cluster_nearby_points(raw_endpoints, tolerance)
    
    # 步骤2：对每个簇，计算精确交点
    refined_points = {}
    for cluster_id, point_list in clusters.items():
        if len(point_list) == 1:
            # 孤立点，保持原样
            refined_points[cluster_id] = point_list[0]
        else:
            # 多条杆件汇聚：求最小二乘交点
            intersection = compute_least_squares_intersection(point_list)
            refined_points[cluster_id] = intersection
    
    # 步骤3：将每条杆件的端点映射到精化后的点
    refined_coords = {}
    for gid, seg in raw_coordinates.items():
        p1_cluster = find_cluster_id(seg[0], clusters, tolerance)
        p2_cluster = find_cluster_id(seg[1], clusters, tolerance)
        
        refined_coords[gid] = [
            refined_points[p1_cluster],
            refined_points[p2_cluster]
        ]
    
    return refined_coords

def compute_least_squares_intersection(point_list):
    """
    多条杆件在同一点汇聚时，求最佳拟合交点
    
    使用最小二乘法求解：
    min Σ ||杆i × 点p||² 
    """
    # 构建线性方程组 A @ p = b
    A = []
    b = []
    
    for point in point_list:
        # 以该点周围的杆件为约束
        neighboring_segs = find_segments_near(point)
        for seg in neighboring_segs:
            # 点到直线的距离约束
            # |ax + by + c| = 0 (归一化)
            a = seg[1][1] - seg[0][1]  # dy
            b_coeff = seg[0][0] - seg[1][0]  # -dx
            c = seg[1][0]*seg[0][1] - seg[0][0]*seg[1][1]
            
            norm = math.sqrt(a**2 + b_coeff**2)
            A.append([a/norm, b_coeff/norm])
            b.append(-c/norm)
    
    # 最小二乘求解
    p = np.linalg.lstsq(A, b, rcond=None)[0]
    return tuple(p)
```

#### 2.5.3 坐标变换链与精度控制

```python
class CoordinateTransformationPipeline:
    """
    坐标处理流水线，确保精度与一致性
    """
    
    def __init__(self):
        self.transforms = []  # 记录所有变换以便回溯
        self.tolerance = 1e-6
    
    def add_transform(self, name, func, inverse_func=None):
        """注册一个变换"""
        self.transforms.append({
            "name": name,
            "forward": func,
            "inverse": inverse_func
        })
    
    def apply_forward(self, coords):
        """顺序应用所有变换"""
        result = coords
        for transform in self.transforms:
            result = transform["forward"](result)
            self._validate(result, transform["name"])
        return result
    
    def apply_inverse(self, coords):
        """逆序应用反变换"""
        result = coords
        for transform in reversed(self.transforms):
            if transform["inverse"]:
                result = transform["inverse"](result)
                self._validate(result, transform["name"] + " (inverse)")
        return result
    
    def _validate(self, coords, stage_name):
        """在每一步检验数据有效性"""
        for gid, seg in coords.items():
            for p in seg:
                for coord in p:
                    if not math.isfinite(coord):
                        raise ValueError(
                            f"非有限值在 {stage_name}: {gid} = {coord}"
                        )
                    if abs(coord) > 1e8:
                        warn(f"极端值在 {stage_name}: {gid} = {coord}")

# 构建完整流水线
pipeline = CoordinateTransformationPipeline()

# 步骤1：数据清洗（移除零长杆件、浮点化）

# 步骤2：精细化端点（聚类、交点计算）

    "refine_endpoints",

# 步骤3：几何约束应用（支撑斜率匹配、横杆投影）

    "apply_constraints",
    apply_geometric_constraints,
```

```python
import logging

logger = logging.getLogger(__name__)

def safe_reconstruction(front_2d, right_2d):
    """
    带容错的3D重建，捕获并报告错误
    """
    try:
        # 前置检查
        if not front_2d or not right_2d:
            logger.warning("输入为空")
            return None
        
        # 尝试重建
        result_3d = combine_views(front_2d, right_2d)
        
        # 后置验证
        if not validate_3d_model(result_3d):
            logger.error("3D模型验证失败")
            return None
        
        logger.info(f"成功重建3D模型，包含{len(result_3d)}条杆件")
        return result_3d
        
    except ZeroDivisionError as e:
        logger.error(f"数学错误（零除）：{e}")
        return None
    except KeyError as e:
        logger.error(f"数据缺失：{e}")
        return None
    except Exception as e:
        logger.exception(f"未预期的错误：{e}")
        return None
```

## 4 系统工作流程

### 4.1 完整处理流程

```
输入文件读取
    ↓
[自动分类：双视图 vs 单视图]
    ├─→ [A线：双视图处理] ──────┐
    │                           ├─→ [拼接] → [标准化] → [输出]
    └─→ [B线：单视图处理] ──────┘
```

### 4.2 A线（双视图）处理流程

1. **数据加载与清洗**
   - 读取正侧两视图坐标
   - 浮点化、去零长杆件

2. **图元分类**
   - 识别支撑杆、横向杆、X型杆

3. **几何约束应用**
   - 支撑斜率匹配（跨视图）
   - 顶横杆对齐与等长
   - 其他横杆成对统一

4. **X型杆件修正**
   - 配对与匹配
   - 端点对齐到支撑

5. **3D坐标生成**
   - 投影组合
   - 中心化与标准化

6. **节点与杆件生成**
   - 从3D线段端点提取节点
   - 建立拓扑关系

### 4.3 B线（单视图）处理流程

1. **节点分类**
   - 一类节点（真实3D坐标）
   - 二类节点（编码型坐标）

2. **迭代求解**
   - 逐轮解算二类节点
   - 基于参考杆件插值

3. **对称性展开**
   - 根据对称标记生成对称节点

4. **坐标变换**
   - 如需要，应用局部坐标系变换

### 4.4 拼接与融合

1. **特征点识别**
   - A线顶部横杆
   - B线底部横杆

2. **相对位置估计**
   - 特征点匹配
   - 计算变换矩阵

3. **坐标变换与合并**
   - 应用变换到B线
   - 去重与融合

### 4.5 最终处理

1. **节点编号标准化**
   - 确保编号在1-500范围

2. **杆件属性补充**
   - 截面推荐
   - 材质分配

3. **数据验证**
   - 拓扑检查
   - 几何合理性验证

4. **输出生成**
   - Excel 结构化表单
   - JSON 完整参数
   - 3D 可视化模型

## 6 关键技术创新点

### 6.1 跨视图约束融合

创新之处：**跨视图斜率统一与成对横杆约束**

- 不仅在单视图内部进行约束，而是建立双视图间的全局约束
- 通过加权平均或最小二乘法统一支撑杆斜率
- 确保生成的3D模型满足两视图的投影一致性

### 6.2 迭代二类节点求解

创新之处：**多轮迭代+依赖图管理**

- 支持二类节点相互引用的复杂情况
- 自动检测求解顺序依赖，避免死循环
- 提供回退机制处理无解情况

### 6.3 自适应几何修正

创新之处：**基于拓扑关系的端点精细化**

- 在聚类基础上计算最小二乘交点
- 自动识别多条杆件汇聚的关键点
- 消除扫描误差带来的坐标偏离

### 6.4 多模型智能拼接

创新之处：**特征点匹配+变换估计**

- 自动识别塔体与塔头的拼接界面
- 通过特征点匹配计算相对位置关系
- 支持非重合、有间隙的拼接情况

```python
    """计算模型验证指标"""

        "node_count": len(nodes),
```
