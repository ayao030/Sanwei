## 2 核心算法的伪代码与注释

### 2.1 支撑杆识别算法

**输入**：view_coords 字典，包含所有杆件坐标

**输出**：支撑杆子集

```pseudocode
算法：find_supports(view_coords)
  
  候选集合 ← []
  对每条线段 (gid, seg) 在 view_coords:
    ID字符串 ← convert_to_string(gid)
    
    // 检查ID格式
    如果 ID字符串.仅含数字 AND
        ID字符串长度 >= 2 AND
        ID字符串最后两位 ∈ {"01", "02", "03", "04"} AND
        ID字符串不含下划线:
      候选集合.添加(gid, seg)
  
  // 按长度过滤
  如果 候选集合为空:
    返回 {}
  
  长度字典 ← {}
  对每条线段 (gid, seg) 在 候选集合:
    长度字典[gid] ← compute_length(seg[0], seg[1])
  
  最大长度 ← max(长度字典.values())
  长度阈值 ← 最大长度 × 0.6
  
  结果 ← {}
  对每条线段 (gid, seg) 在 候选集合:
    如果 长度字典[gid] >= 长度阈值:
      结果[gid] ← seg
  
  返回 结果 或 候选集合  // 如果过滤太严格则返回所有候选

状态：O(n log n) 时间复杂度
```

### 2.2 支撑斜率统一算法

**输入**：front_supports, right_supports（两视图的支撑杆）

**输出**：统一斜率后的支撑杆坐标

```pseudocode
算法：match_support_slopes(front_supports, right_supports)
  
  // 步骤1：计算各杆件斜率
  front_slopes ← []
  对每条杆件 seg 在 front_supports.values():
    斜率 ← (seg[1].y - seg[0].y) / (seg[1].x - seg[0].x + ε)
    front_slopes.添加(斜率)
  
  right_slopes ← []
  对每条杆件 seg 在 right_supports.values():
    斜率 ← (seg[1].y - seg[0].y) / (seg[1].x - seg[0].x + ε)
    right_slopes.添加(斜率)
  
  // 步骤2：计算统一斜率（加权平均）
  所有斜率 ← front_slopes + right_slopes
  
  // 移除离群值（median ± 3σ）
  中位数 ← median(所有斜率)
  标准差 ← std(所有斜率)
  有效斜率 ← filter(|k - 中位数| <= 3×标准差)
  
  统一斜率 ← mean(有效斜率)
  
  // 步骤3：重新调整端点
  对每条杆件 (gid, seg) 在 front_supports.items():
    中点 ← ((seg[0].x + seg[1].x)/2, (seg[0].y + seg[1].y)/2)
    长度 ← compute_length(seg[0], seg[1])
    
    // 保持长度，用新斜率计算端点
    dx ← 长度 / √(1 + 统一斜率²)
    dy ← 统一斜率 × dx
    
    新_p1 ← (中点.x - dx/2, 中点.y - dy/2)
    新_p2 ← (中点.x + dx/2, 中点.y + dy/2)
    
    front_supports[gid] ← [新_p1, 新_p2]
  
  // 右视图同理...
  
  返回 (front_supports, right_supports)

状态：O(n log n) 时间，统计鲁棒
```

### 2.3 顶横杆定位与等长约束

**输入**：支撑杆模型、原始横向杆

**输出**：调整后的顶横杆坐标

```pseudocode
算法：align_top_span(front_support_models, right_support_models,
                      front_horizontals, right_horizontals)
  
  // 步骤1：找出两视图支撑杆的最高点
  y_top_f ← max({seg[0].y, seg[1].y | seg ∈ front_support_models})
  y_top_r ← max({seg[0].y, seg[1].y | seg ∈ right_support_models})
  y_top_unified ← (y_top_f + y_top_r) / 2
  
  // 步骤2：在该高度获取左右极值坐标
  对于高度 y_top_unified:
    // 在正面图中
    left_f ← min_x_at_y(front_support_models, y_top_unified)
    right_f ← max_x_at_y(front_support_models, y_top_unified)
    
    // 在侧面图中
    left_r ← min_x_at_y(right_support_models, y_top_unified)
    right_r ← max_x_at_y(right_support_models, y_top_unified)
  
  // 步骤3：定位顶横杆
  // 在正面图中找到最高的横向杆
  top_h_front_key ← argmax({avg_y(seg) | seg ∈ front_horizontals.values()})
  top_h_right_key ← argmax({avg_y(seg) | seg ∈ right_horizontals.values()})
  
  // 更新坐标
  front_horizontals[top_h_front_key] ← [(left_f, y_top_unified), (right_f, y_top_unified)]
  right_horizontals[top_h_right_key] ← [(left_r, y_top_unified), (right_r, y_top_unified)]
  
  // 步骤4：验证长度相等性
  len_f ← distance(left_f, right_f)
  len_r ← distance(left_r, right_r)
  
  如果 |len_f - len_r| / max(len_f, len_r) > 0.05:
    警告: "顶横杆长度不等，差异 " + str(|len_f - len_r|)
  
  返回 (front_horizontals, right_horizontals)

状态：O(n) 时间，几何精确度依赖支撑模型质量
```

### 2.4 二类节点迭代求解

**输入**：nodes_raw（混合的一类和二类节点）

**输出**：完整的3D节点坐标

```pseudocode
算法：resolve_all_class2_nodes(nodes_raw)
  
  // 步骤0：初始化
  class1_nodes ← extract_nodes_where(node_type == 11, nodes_raw)
  node_map ← {node.node_id: node | node ∈ class1_nodes}
  unresolved ← extract_nodes_where(node_type == 12, nodes_raw)
  
  迭代次数 ← 0
  最大迭代 ← 10
  
  // 步骤1：迭代求解
  当 unresolved非空 AND 迭代次数 < 最大迭代:
    迭代次数 ← 迭代次数 + 1
    刚解决的 ← []
    
    对于每个节点 node 在 unresolved:
      // 解析节点的编码
      real_dim, real_value, ref_id1, ref_id2 ← parse_class2(node)
      
      // 检查参考节点是否已知
      如果 ref_id1 不在 node_map OR ref_id2 不在 node_map:
        继续  // 跳过，依赖未满足
      
      // 线性插值
      p1 ← node_map[ref_id1]
      p2 ← node_map[ref_id2]
      
      结果 ← try_linear_interpolation(real_dim, real_value, p1, p2)
      
      如果 结果 非 NULL:
        // 生成新节点
        new_node ← {
          node_id: node.node_id,
          X: 结果.x,
          Y: 结果.y,
          Z: 结果.z,
          node_type: 11
        }
        node_map[node.node_id] ← new_node
        刚解决的.添加(node)
    
    // 步骤2：检查进度
    如果 刚解决的.为空:
      警告: "第 " + str(迭代次数) + " 轮未有进展，停止迭代"
      break
    
    // 步骤3：更新未解决列表
    unresolved ← [n for n in unresolved if n 不在 刚解决的]
  
  // 步骤4：返回所有节点
  返回 list(node_map.values())

状态：最坏 O(n²) 时间（最多n轮，每轮O(n)），通常更快
```

### 2.5 坐标中心化与平移

**输入**：双视图3D坐标模型

**输出**：中心化后的3D坐标

```pseudocode
算法：center_and_translate_3d(coords_3d_front, coords_3d_right)
  
  // 步骤1：收集所有端点
  所有点 ← []
  对于 seg 在 coords_3d_front.values():
    所有点.添加(seg[0])
    所有点.添加(seg[1])
  对于 seg 在 coords_3d_right.values():
    所有点.添加(seg[0])
    所有点.添加(seg[1])
  
  // 步骤2：计算极值与中心
  x_min ← min({p.x | p ∈ 所有点})
  x_max ← max({p.x | p ∈ 所有点})
  y_min ← min({p.y | p ∈ 所有点})
  y_max ← max({p.y | p ∈ 所有点})
  z_min ← min({p.z | p ∈ 所有点})
  z_max ← max({p.z | p ∈ 所有点})
  
  x_center ← (x_min + x_max) / 2
  y_center ← (y_min + y_max) / 2
  
  // 步骤3：计算平移向量
  // 目标：X,Y 中心化为0；Z 最低点在0
  dx ← -x_center
  dy ← -y_center
  dz ← -z_min
  
  // 步骤4：应用平移
  def apply_translation(coords_dict):
    return {
      gid: [(p.x+dx, p.y+dy, p.z+dz) for p in seg]
      for gid, seg in coords_dict.items()
    }
  
  translated_front ← apply_translation(coords_3d_front)
  translated_right ← apply_translation(coords_3d_right)
  
  // 步骤5：返回
  center_info ← {
    original_center: (x_center, y_center, 0),
    z_min: z_min,
    translation_vector: (dx, dy, dz)
  }
  
  返回 (translated_front, translated_right, center_info)

状态：O(n) 时间和空间，线性复杂度
```

```python
def validate_node(node):
    """验证节点数据的有效性"""
    
    # 必填字段
    assert 'node_id' in node, "缺少 node_id"
    assert 'X' in node and 'Y' in node and 'Z' in node, "缺少坐标字段"
    
    node_type = node.get('node_type', 11)
    node_id = int(str(node['node_id'])[:-1])  # 提取前缀数字
    
    # 节点ID有效性
    if 1 <= node_id <= 500:
        pass  # 合法范围
    else:
        warn(f"节点ID {node_id} 超出标准范围 [1, 500]")
    
    # 节点类型判断
    if node_type == 11:
        # 一类节点：所有坐标都应该是浮点数
        for coord in ['X', 'Y', 'Z']:
            val = node[coord]
            assert isinstance(val, (int, float)), f"{coord} 应为数值"
            assert math.isfinite(val), f"{coord} 包含NaN或Inf"
            assert -1e6 < val < 1e6, f"{coord} 超出合理范围"
    
    elif node_type == 12:
        # 二类节点：应有2个编码坐标和1个真实坐标
        encoded_count = sum(
            1 for v in [node['X'], node['Y'], node['Z']]
            if isinstance(v, int) and v >= 10000
        )
        assert encoded_count == 2, f"二类节点应有2个编码坐标，实际{encoded_count}个"
    
    # 对称性有效性
    sym = node.get('symmetry_type', 0)
    assert sym in [0, 1, 2, 3, 4], f"无效的对称性类型: {sym}"
    
    return True
```

```python
def validate_member(member, nodes):
    """验证杆件的拓扑和几何有效性"""
    
    # 必填字段
    assert 'member_id' in member, "缺少 member_id"
    assert 'node1_id' in member and 'node2_id' in member, "缺少节点引用"
    
    # 节点存在性
    n1_id = str(member['node1_id'])
    n2_id = str(member['node2_id'])
    assert n1_id in nodes, f"起点节点 {n1_id} 不存在"
    assert n2_id in nodes, f"终点节点 {n2_id} 不存在"
    
    # 自循环检查
    assert n1_id != n2_id, f"杆件 {member['member_id']} 自循环"
    
    # 长度有效性
    n1 = nodes[n1_id]
    n2 = nodes[n2_id]
    
    dist = math.sqrt(
        (n2['X'] - n1['X'])**2 +
        (n2['Y'] - n1['Y'])**2 +
        (n2['Z'] - n1['Z'])**2
    )
    
    assert dist > 1e-6, f"杆件长度过小: {dist:.2e}"
    assert dist < 1000, f"杆件长度过大: {dist}"
    
    # 如果指定了长度，应该匹配
    if 'length' in member:
        assert abs(member['length'] - dist) < 0.01, \
            f"长度不匹配: 期望{member['length']}, 实际{dist:.3f}"
    
    return True
```

## 4 误差分析与容差设置

### 4.1 主要误差来源

| 误差来源 | 典型大小 | 影响 | 处理方法 |
|---------|---------|------|--------|
| 扫描误差 | ±1-2 mm | 坐标偏离 | 聚类+交点精化 |
| 浮点舍入 | 1e-10 | 判等失效 | 使用 isclose() |
| 手工输入错误 | ±5-10 mm | 系统性偏移 | 异常检测 |
| 倾斜拍摄 | ±2-3% | 比例失真 | 几何约束修正 |
| 数据不一致 | ±5-20 mm | Z轴偏差 | 平均或加权选择 |

### 4.3 异常检测与处理

```python
def detect_data_anomalies(coords_2d):
    """检测2D坐标数据中的异常"""
    
    anomalies = {
        'zero_length_segments': [],
        'extremely_long_segments': [],
        'isolated_points': [],
        'duplicate_ids': [],
        'coordinate_range_issues': []
    }
    
    # 统计长度分布
    lengths = [
        euclidean_distance(seg[0], seg[1])
        for seg in coords_2d.values()
    ]
    
    if not lengths:
        return anomalies
    
    avg_length = statistics.mean(lengths)
    stdev_length = statistics.stdev(lengths) if len(lengths) > 1 else 0
    
    # 检查各线段
    for gid, seg in coords_2d.items():
        length = euclidean_distance(seg[0], seg[1])
        
        # 零长线段
        if length < 1e-6:
            anomalies['zero_length_segments'].append((gid, length))
        
        # 极长线段
        if length > avg_length + 5 * stdev_length:
            anomalies['extremely_long_segments'].append((gid, length))
        
        # 坐标范围
        for i, (x, y) in enumerate(seg):
            if not (-1e6 < x < 1e6 or -1e6 < y < 1e6):
                anomalies['coordinate_range_issues'].append((gid, i, (x, y)))
    
    # 检查重复ID
    id_counts = {}
    for gid in coords_2d:
        id_counts[gid] = id_counts.get(gid, 0) + 1
    
    for gid, count in id_counts.items():
        if count > 1:
            anomalies['duplicate_ids'].append((gid, count))
    
    return anomalies

def handle_anomalies(anomalies, strategy='warn'):
    """根据策略处理异常"""
    
    if strategy == 'warn':
        if anomalies['zero_length_segments']:

            warn(f"发现 {len(anomalies['duplicate_ids'])} 个重复ID")
    
    elif strategy == 'remove':
        # 移除零长线段
        # 移除重复ID（保留第一个）
        pass
    
    elif strategy == 'error':
        if anomalies['duplicate_ids']:
            raise ValueError(f"存在重复ID: {anomalies['duplicate_ids']}")
```

## 5 性能优化与大规模处理

### 5.1 计算复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 | 备注 |
|------|-----------|-----------|------|
| 数据读取 | O(n) | O(n) | n=杆件数量 |
| 图元分类 | O(n log n) | O(n) | 排序+分类 |
| 支撑匹配 | O(n log n) | O(n) | 加权平均+离群值 |
| X型配对 | O(n log n) | O(n) | 排序+匹配 |
| 3D投影 | O(n) | O(n) | 直接组合 |
| 二类求解 | O(n²) 最坏 | O(n) | 迭代，通常<10轮 |
| 拼接 | O(m+n) | O(m+n) | 两个模型合并 |
| **总体** | **O(n²)** | **O(n)** | 通常n<2000 |

### 5.2 优化策略

**1. 缓存常用计算**：

```python
class ComputationCache:
    """缓存中间计算结果，避免重复"""
    
    def __init__(self):
        self.cache = {}
    
    def compute_length(self, seg_id, seg):
        if seg_id in self.cache:
            return self.cache[seg_id]
        
        length = euclidean_distance(seg[0], seg[1])
        self.cache[seg_id] = length
        return length
    
    def clear(self):
        self.cache.clear()
```

```python
from scipy.spatial import KDTree

class SpatialIndex:
    """使用KD树加速空间查询"""
    
    def __init__(self, nodes):
        self.nodes = nodes
        points = np.array([
            [n['X'], n['Y'], n['Z']] for n in nodes.values()
        ])
        self.kdtree = KDTree(points)
    
    def find_nearby(self, query_point, radius=1.0):
        """查询半径内的节点"""
        indices = self.kdtree.query_ball_point(query_point, radius)
        return [list(self.nodes.values())[i] for i in indices]
    
    def find_nearest(self, query_point, k=1):
        """查询最近的k个节点"""
        distances, indices = self.kdtree.query(query_point, k=k)
        return [list(self.nodes.values())[i] for i in indices]
```

## 7 常见问题与故障排除

### 7.1 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|-----|------|--------|
| "KeyError: node not found" | 杆件引用了不存在的节点 | 检查节点提取是否完整，确保所有端点都被识别 |
| "Zero-length member" | 两端点重合 | 检查坐标数据，可能存在扫描误差或重复点 |
| "Z值不一致" | 两视图的Z坐标偏差大 | 检查图纸是否水平拍摄，调整容差或采用平均值 |
| "二类节点无法求解" | 参考杆件不存在或无法插值 | 检查编码是否正确，验证参考节点是否已解析 |
| "拼接失败" | 特征点匹配不准确 | 放宽匹配容差，或手动指定拼接点 |

## 8 性能优化建议

### 8.1 内存优化

- **流式处理**：对大型文件采用分块读取而非一次性加载
- **及时释放**：处理完中间数据后及时删除，释放内存
- **数据压缩**：使用更紧凑的数据格式存储

### 8.2 速度优化

- **并行化**：多线程或多进程处理独立的杆件
- **向量化**：使用NumPy批量操作而非Python循环
- **缓存**：缓存重复计算（如杆件长度、斜率）

### 8.3 精度优化

- **高精度算术**：关键计算使用double而非float
- **稳定算法**：使用数值稳定的方法（如正交分解）
- **验证机制**：多步验证确保精度
